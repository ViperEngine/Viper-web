ViperEngine Documentation

Welcome to the official documentation for ViperEngine, a minimal yet powerful 2D game development framework inspired by Pygame but built on SDL2 and Python's ctypes for performance and flexibility.

üöÄ Getting Started

To install ViperEngine, simply run:

pip install ViperEngine

Make sure you have Python 3.6 or higher installed on your system.

Once installed, you can begin building your game using the familiar API patterns inspired by Pygame, but with lower-level control.

üß± Quick Start Example

Here is a minimal example that opens a window, listens for the quit event, and updates the screen at 60 FPS:

import viperengine as ve

ve.init()  # Initialize the engine and SDL2 subsystems

screen = ve.Display().set_mode((640, 480))  # Create a window of 640x480 pixels
screen.set_caption("Hello Viper")          # Set the window title

clock = ve.Clock()  # Create a clock object to manage FPS

running = True
while running:
    screen.paint((0, 0, 0))  # Clear screen with black color

    # Poll for input events (keyboard, mouse, window)
    for event in ve.poll_event() or []:
        if event["type"] == "QUIT":
            running = False  # Exit the loop if the window is closed

    screen.update()   # Update the window with new frame contents
    clock.tick(60)    # Limit loop to 60 frames per second

ve.quit()  # Cleanly shut down the engine

This example covers the basic game loop pattern, event polling, drawing, and frame rate control.

üìö API Overview

Here‚Äôs a quick rundown of the main classes and functions you will use in ViperEngine:

  • Display: Represents the game window. Use set_mode() to set size, set_caption() to change title, paint() to fill the screen with color, and update() to refresh the display.
  • Rect: Utility class for rectangular shapes. Useful for positioning, collision detection, and managing sprite bounds.
  • Surface: Represents images and textures. Load images from files, draw them on the screen, and manipulate pixels.
  • poll_event(): Polls SDL2 events such as keyboard input, mouse clicks, and window actions. Returns a list of event dictionaries.
  • Clock: Controls the game loop timing, allowing you to cap the frames per second using tick(fps).

üé® Drawing and Rendering

ViperEngine lets you draw basic shapes and images. For example:

screen.paint((255, 255, 255))  # Fill screen with white
screen.draw_rect((255, 0, 0), (50, 50, 100, 100))  # Draw red rectangle
surface = ve.Surface("player.png")  # Load an image
screen.blit(surface, (200, 150))    # Draw the image at (200,150)
screen.update()

Events are returned as dictionaries with a type field (e.g. KEYDOWN, QUIT). For example, handle keyboard input:

for event in ve.poll_event() or []:
    if event["type"] == "KEYDOWN":
        if event["key"] == "ESCAPE":
            running = False
